# Auto Trading System

> 한국 주식 시장 스윙 트레이딩 자동매매 시스템.
> LLM과 정량 룰을 결합하여 15년차 직장인 트레이더의 매일 작업을 자동화한다.

## 프로젝트 정체성 (한 문단)

종목 직관 없이 메타 직관을 가진 시스템 빌더가 한국 시장의 비효율(외인/기관 일별 수급 공개, 한국어 NLP 틈새, 알파 부패 느림)을 노려 만드는 자동매매 시스템. **시스템은 예측하지 않고 모든 시나리오에 사전 대응한다.** LLM은 재료 해석·시황 판단·뉴스 trust_score에만 쓰고 매매 결정은 정량 룰. Multi-tier 트리거(수급→뉴스→가격)로 시장 인과 순서대로 신호를 잡고, 매크로 충격은 분할 진입으로 대응. AI는 분석가·사람은 결정자.

2. **1주차 데이터 파이프라인 범위** — 최소/표준/풀
3. **백테스트 엔진 1차 선택** — vectorbt / backtesting.py / 자체

## 핵심 원칙 (3가지만)

1. **4-layer 분리** — LLM은 Layer 2까지만, 매매 결정은 정량 룰
2. **데이터 3분할** — Train/Validate/Test, Test는 단 1회
3. **예측 ❌, 대응 ⭕** — 모든 시나리오에 사전 행동 정의

## 흔한 LLM 트레이딩 실수 분류 — 이 시스템이 피하는 20가지 함정

학계·업계 LLM 트레이딩 시스템이 반복적으로 빠지는 함정을 5개 그룹으로 분류. 본 시스템의 각 설계 결정은 **어떤 함정을 막기 위한 것인가**로 추적 가능해야 한다.

### Group A — 잘못된 도구 사용 (#1, #2, #3, #20)

> LLM에게 매매·예측·sentiment 단순 작업을 시키는 것

LLM은 텍스트 패턴(뉴스·시황·재료)에 강하고, 시계열·수급·호가창 같은 정량 매매 결정에는 약함. 학계 LLM 트레이딩 연구가 실거래에서 무너지는 이유 (FINSABER 2025).

**본 시스템 해결:**
- LLM은 **분석가**, 매매는 정량 룰 (`PROJECT_GUIDELINES.md` §3.2)
- LLM 출력 = 항상 정량 점수 + 태그, 자유 텍스트 매매 결정 ❌
- LLM 정성 분석 한계 명문화 (메타 추론·권력 동학·비공개 정보·반사실적 추론) (§3.3)
- BUY/SELL 직접 결정 / 포지션 크기 / 손절 라인 / 룰 자동 변경 모두 LLM에게 ❌

### Group B — 시장 메커니즘 무시 (#4, #5, #6)

> 둔감화 / 인과 순서 / trust / **선반영-기대감 해소**를 무시하는 것

시장은 "결과"가 아니라 "예상 대비 차이"를 반영하고, 같은 재료는 시간 갈수록 둔감해지고, 가격은 결과·수급/뉴스가 원인이며, 모든 뉴스가 동일 가중치를 받지 않는다. 그리고 **기대감과 불안감 자체가 가격에 미리 반영(선반영)되어, 이벤트 발생 시 surprise = 0 인 경우에도 가격이 움직인다.**

**본 시스템 해결 — 시장 작동 방식을 그대로 시스템에 박음:**
- §3.9 Surprise & Freshness 원칙 — 단순 sentiment ❌, 예상 대비 surprise + 재료 freshness 결합
- §3.12 선반영 + 기대감 해소 메커니즘 — "Buy the rumor, sell the news" 그리고 그 역(Buy the fear-resolution) 시스템에 명문화
- §9.3 Multi-Tier 트리거 — 수급(원인) → 뉴스(공식화) → 가격(결과) 인과 순서
- 뉴스 trust_score 시스템 — 매체 신뢰도 / 단독 vs 복수 / 시간 경과 / 작전 패턴 감지

### Group C — 통계 방법론 부재 (#7, #8, #15)

> Anchor 없음 / 검증 없음 / Test 오염

흔들리는 fuzzy 조건으로 통계내기, 조건 강화했는데 승률 안 올라도 채택, Test set 보고 전략 수정 — Bailey & López de Prado가 *"Probability of Backtest Overfitting"* 으로 정량화한 함정.

**본 시스템 해결 — 통계학적 엄밀성:**
- §3.10 Anchor Event 원칙 — 모든 통계는 *"흔들리지 않는 기준"* 위에서 시작
- §4.1 데이터 3분할 절대 원칙 — Test는 **단 1회**, 보고 전략 고치면 그 순간 Train으로 변환
- §4.6 Feature 검증 룰 (Win Rate Lift + IC 단조성) — 조건 강화 → 측정 지표 ↑ 안 되면 그 조건은 알파 ❌
- 표본 크기 N < 100인 통계로 slot 채택 ❌

### Group D — 리스크 관리 실패 (#9, #10, #11, #14)

> 단일 알파 / 풀매수 / 손절 못함 / 거래비용 무시

깡통의 직접적 원인. McLean & Pontiff (2016) 알파 부패 -58%, Chen & Velikov (2022) 거래비용 후 -93% — 단일 알파 의존은 통계적으로 죽음.

**본 시스템 해결:**
- §7.2 다층 alpha slot 시스템 — 단일 알파 ❌, slot별 등장빈도/손익비/자본활용률 매월 측정
- §6.4 분할 진입 (1차 5% / 2차 3% / 3차 2%) — 매크로 충격은 예측 ❌, 분할 진입으로 대응 ⭕
- §6.3 킬스위치 3종 (하드코딩, 런타임 변경 ❌) — 종목별 손절·일일 -3%·누적 -15%
- §6.4 거래비용 다단계 시뮬레이션 — Default 0.5%, stress test 0.8%
- §6.8 비중 정책 — 종목당 10%, 동시 5~10종목, 시드 100% 활용 강제 ❌

### Group E — 인간 심리 함정 (#12, #13, #16, #17, #18, #19)

> 개입 / 과로 / AI 환상 / 데이트레이딩 / 미국 따라가기 / 섹터 무시

Barber & Odean (2000): 거래 빈도 상위 20%는 하위 20%보다 **연 7%p 낮은 수익**. 본인 개입이 본인을 갉아먹음. 처분 효과(Shefrin & Statman 1985), 손실 회피, 최근성 편향, 확증 편향이 누적된 결과.

**본 시스템 해결:**
- §9.4 모드별 비대칭 권한 — 보수 방향(축소·청산·중단)만 허용, 공격 방향(추가 매수·손절 풀기) ❌
- §6.6 일일 운영 리듬 — 새벽 시나리오 / 8:30 승인 / 장중 무개입 / 저녁 회고
- §8 자기 개선의 한계 — AI는 분석가, 사람은 결정자, 시스템은 결정론적 룰 실행
- §1 사용자 프로필 — 스윙 (며칠~몇 주), 데이트레이딩 ❌, 직장인 일과 그대로
- §2 시장 선택 근거 — 한국 시장 우위 명확 (자동매매 = 한국, 미국은 1년+ 후 옵션)
- §4.7 섹터 로테이션 (Regime × Sector 매트릭스) — 전체 시장 방향 ❌, 섹터별 차등
