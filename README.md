# Auto Trading System

> 한국 주식 시장 스윙 트레이딩 자동매매 시스템.
> LLM과 정량 룰을 결합하여, 검증된 트레이더의 매일 작업을 대학생 시스템 빌더가 자동화한다.

## 프로젝트 정체성 (한 문단)

종목 직관 없이 메타 직관을 가진 대학생 시스템 빌더가 한국 시장의 비효율(외인/기관 일별 수급 공개, 한국어 NLP 틈새, 알파 부패 느림)을 노려 만드는 자동매매 시스템. 검증된 트레이더의 6년 발화에서 조건을 추출해 라이브러리를 만들고, 백테스트로 검증하면서 **영원히 진화시킨다**. **시스템은 예측하지 않고 모든 시나리오에 사전 대응한다.** LLM은 재료 해석·시황 판단·뉴스 trust_score에만 쓰고 매매 결정은 정량 룰. **LLM 출력은 ordinal 등급 분류만 (확률 ❌).** Multi-tier 트리거(수급→뉴스→가격)로 시장 인과 순서대로 신호를 잡고, 매크로 충격은 분할 진입으로 대응. **단일 신호 정점 ❌, 다요소 결합의 분포 비대칭 ⭕** (4요소: 기술적 통계 + 재료 해석 + 시장 심리 + AI 판단). AI는 분석가·사람은 결정자.

## 시스템 빌드 단계 (기한 ❌)

**6주 로드맵은 v3.6에서 폐기됨** — 본 시스템은 프로젝트가 아니라 시스템이므로 마감이 없다 (§0.2, §12). 아래는 *의존 순서*일 뿐 deadline 아님.

| 단계 | 작업 | 상태 |
|---|---|---|
| 1 | 데이터 파이프라인 (가격·수급 + point-in-time 종목 마스터) | 착수 |
| 2 | 백테스트 엔진 + universe filter baseline = **Backtest Phase 1** (정량만) | 착수 |
| 3 | LLM 신호 모듈 (유튜브 자막 → 종목 ordinal 등급) | 대기 |
| 4 | 신호 + 매매 룰 통합 = **Backtest Phase 2** (Phase 1 대비 분포 개선 입증) | 대기 |
| 5 | 시나리오 플레이북 + 페이퍼 트레이딩 | 대기 |
| 6 | 실거래 Phase A (100만원) = forward test | 대기 |

Backtest Phase 1/2(신호 검증)와 자본 투입 Phase A/B/C는 **다른 차원**이다 (§3.16). 시장 진화 → 시스템도 영원히 진화 (§12).

## 핵심 원칙 (3가지만)

1. **4-layer 분리** — LLM은 Layer 2까지만, 매매 결정은 정량 룰
2. **데이터 3분할** — Train/Validate/Test, Test는 단 1회
3. **예측 ❌, 대응 ⭕** — 모든 시나리오에 사전 행동 정의

자세한 내용은 `docs/PROJECT_GUIDELINES.md` 참고.

## 흔한 LLM 트레이딩 실수 분류 — 이 시스템이 피하는 20가지 함정

학계·업계 LLM 트레이딩 시스템이 반복적으로 빠지는 함정을 5개 그룹으로 분류. 본 시스템의 각 설계 결정은 **어떤 함정을 막기 위한 것인가**로 추적 가능해야 한다.

### Group A — 잘못된 도구 사용 (#1, #2, #3, #20)

> LLM에게 매매·예측·sentiment 단순 작업을 시키는 것

LLM은 텍스트 패턴(뉴스·시황·재료)에 강하고, 시계열·수급·호가창 같은 정량 매매 결정에는 약함. 학계 LLM 트레이딩 연구가 실거래에서 무너지는 이유 (FINSABER 2025).

**본 시스템 해결:**
- LLM은 **분석가**, 매매는 정량 룰 (`PROJECT_GUIDELINES.md` §3.2)
- LLM 출력 = 항상 **ordinal 등급 + 태그**, 자유 텍스트·정량 확률 매매 결정 ❌ (v3.6)
- LLM 정성 분석 한계 명문화 (메타 추론·권력 동학·비공개 정보·반사실적 추론) (§3.3)
- BUY/SELL 직접 결정 / 포지션 크기 / 손절 라인 / 룰 자동 변경 모두 LLM에게 ❌
- **재료 5축 분해 (§3.9.x) + 반전성 (§4.4.x) + 통제성 + 직접성** 모두 LLM ordinal로 분해 (v3.7 ⭐)
- **MAX 플랜 batch 운용** (§5.5) — 토큰비 회피, Claude Code/브라우저 채팅 (v3.7 ⭐)

### Group B — 시장 메커니즘 무시 (#4, #5, #6)

> 둔감화 / 인과 순서 / trust / **선반영-기대감 해소** / **반전 메커니즘**을 무시하는 것

시장은 "결과"가 아니라 "예상 대비 차이"를 반영하고, 같은 재료는 시간 갈수록 둔감해지고, 가격은 결과·수급/뉴스가 원인이며, 모든 뉴스가 동일 가중치를 받지 않는다. **기대감과 불안감 자체가 가격에 미리 반영(선반영)되어, 이벤트 발생 시 surprise = 0 인 경우에도 가격이 움직인다.** 그리고 **반전(시장 컨센서스 정반대 발생) 자체가 시장이 가장 좋아하는 키워드.** ⭐ v3.7

**본 시스템 해결 — 시장 작동 방식을 그대로 시스템에 박음:**
- §3.9 Surprise & Freshness 원칙 — 단순 sentiment ❌, 예상 대비 surprise + 재료 freshness 결합
- §3.9.x 재료 5축 분해 — 수혜금액/반사이익/상징성/세계최초/M&A (v3.7 ⭐)
- §3.12 선반영 + 기대감 해소 메커니즘 — "Buy the rumor, sell the news" 그리고 그 역(Buy the fear-resolution) 시스템에 명문화
- §4.4.x 반전성 — 4가지 타입 (anticipation_resolved / concern_dissolved / phoenix_revival / consensus_reversal) (v3.7 ⭐)
- §9.3 Multi-Tier 트리거 — 수급(원인) → 뉴스(공식화) → 가격(결과) 인과 순서
- §9.3 Tier 1에 양매수 5축 detection (F10) + 거래원 broker flow (F8/F9) 통합 (v3.7 ⭐)
- 뉴스 trust_score 시스템 — 매체 신뢰도 / 단독 vs 복수 / 시간 경과 / 작전 패턴 감지
- final_signal_strength = trust × surprise × freshness × (1−discharge) × (1−leak) × **reversal_amplifier** (6차원 곱) (v3.7 ⭐)

### Group C — 통계 방법론 부재 (#7, #8, #15)

> Anchor 없음 / 검증 없음 / Test 오염 / **단일 신호 정점만 보기**

흔들리는 fuzzy 조건으로 통계내기, 조건 강화했는데 승률 안 올라도 채택, Test set 보고 전략 수정 — Bailey & López de Prado가 *"Probability of Backtest Overfitting"* 으로 정량화한 함정. 그리고 **단일 신호 정점만 보면 노이즈에 휘말림 (Tversky 1972 다속성 의사결정).** ⭐ v3.7

**본 시스템 해결 — 통계학적 엄밀성 + 결합 신호:**
- §3.10 Anchor Event 원칙 — 모든 통계는 *"흔들리지 않는 기준"* 위에서 시작
- §3.13 Modular Backtesting — Layer별 단독 백테스트 + drop-one validation (v3.6)
- §3.14 시간 자체가 자원 — Forward test = 시간 누적 (v3.6)
- §3.16 결합 신호 원리 — 정보 0 = 50% 승률, 결합할수록 ↑ + Backtest Phase 1/2 분리 (v3.7 ⭐)
- §4.1 데이터 3분할 절대 원칙 — Test는 **단 1회**, 보고 전략 고치면 그 순간 Train으로 변환
- §4.6 Feature 검증 룰 (Win Rate Lift + IC 단조성) — 조건 강화 → 측정 지표 ↑ 안 되면 그 조건은 알파 ❌
- 표본 크기 N < 100인 통계로 slot 채택 ❌
- §6.2 Conjunctive + Compensatory 이중 게이트 + §4.9 회피 패턴 Hard Filter (v3.7) ⭐
- §6.1.x 손익비 비대칭 3 lever (WR/AW/AL) — 메인 lever 사후 측정 (v3.7 ⭐)
- §6.1.y Backtest Phase 1 vs 2 비교로 LLM 도입 가치 정량 입증 (v3.7 ⭐)
- §6.4.x 매매일지 자동 분석 7차원 (v3.7 ⭐)

### Group D — 리스크 관리 실패 (#9, #10, #11, #14)

> 단일 알파 / 풀매수 / 손절 못함 / 거래비용 무시

깡통의 직접적 원인. McLean & Pontiff (2016) 알파 부패 -58%, Chen & Velikov (2022) 거래비용 후 -93% — 단일 알파 의존은 통계적으로 죽음.

**본 시스템 해결:**
- §1.4 시간 자녀 마인드 — 매매 안 해도 죽지 않음, 미수/신용/스탁론 ❌ (v3.7 ⭐)
- §7.2 다층 alpha slot 시스템 — 단일 알파 ❌, slot별 등장빈도/손익비/자본활용률 매월 측정
- §6.4 분할 진입 (1차 5% / 2차 3% / 3차 2%) — 매크로 충격은 예측 ❌, 분할 진입으로 대응 ⭕
- §6.3 킬스위치 3종 (하드코딩, 런타임 변경 ❌) — 종목별 손절·일일 -3%·누적 -15%
- §6.4 거래비용 다단계 시뮬레이션 — Default 0.5%, stress test 0.8%
- §6.8 비중 정책 — 종목당 10%, 동시 5~10종목, 시드 100% 활용 강제 ❌
- §4.9 회피 패턴 Hard Filter 6가지 — conviction 무관 진입 ❌ (v3.7 ⭐)
  - 거래대금 후 흐름 4가지 (single_stock / single_event / uncontrollable / dishwash_rebound)
  - 쫄쫄이 우상향 (영풍제지 패턴)
  - 홀짝 매매 (오버나이트)
  - 거래원 매도 집중 (한 창구 30%+)
  - 기타법인 매도 폭증 (z-score > 3)
  - 바이오 + 단발성 재료 (low_rebound)

### Group E — 인간 심리 함정 (#12, #13, #16, #17, #18, #19)

> 개입 / 과로 / AI 환상 / 데이트레이딩 / 미국 따라가기 / 섹터 무시

Barber & Odean (2000): 거래 빈도 상위 20%는 하위 20%보다 **연 7%p 낮은 수익**. 본인 개입이 본인을 갉아먹음. 처분 효과(Shefrin & Statman 1985), 손실 회피, 최근성 편향, 확증 편향이 누적된 결과.

**본 시스템 해결:**
- §0.3 매매 스타일 결정 — **스윙만, 단타 ❌** (v3.7 ⭐)
