## 1. 가격/시총 패널 — `FinanceData/marcap`

일별 전종목 스냅샷. `Date`를 인덱스로, 그날 상장돼 있던 모든 종목이 한 행씩. **상장폐지 종목은
상폐 전까지 자기 날짜에 존재** → 망한 회사가 데이터에 남고(살아남은 것만 보는 착시 = survivorship
bias 방지), 각 시점에 실제 거래되던 종목만 보이는 것(point-in-time universe)을 데이터가 보장.

| 컬럼 | 의미 | 타입 | 백테스트 용도 |
|------|------|------|--------------|
| Date | 날짜 | datetime64[ns] | 시계열 키 |
| Code | 종목코드 | str(6) | 종목 키 — **로더가 `zfill(6)` 보정**(아래 주의) |
| Name | 종목명 | str | 표시용 (가명화 대상, Phase 2) |
| Open/High/Low/Close | 시/고/저/종가 | float64 | 진입가(종가), 변동성 |
| Volume | 거래량(주) | **float64** | — |
| **Amount** | **거래대금(원)** | **float64** | **거래대금 z-score(§4.3), 유동성 필터** |
| Changes | 전일대비 | float64 | 등락 |
| ChangeCode | 등락 기호 | str | — |
| ChangesRatio | 전일대비 등락률(%) | float64 | 등락률 필터(§4.2) |
| **Marcap** | **시가총액(원)** | **float64** | **시총 필터(§4.2)** |
| **Stocks** | **상장주식수** | int64 | 유통 규모 |
| MarketId | 시장기호 | str | — |
| Market | 시장 | str | `KOSPI` / `KOSDAQ` / `KOSDAQ GLOBAL` / `KONEX` |
| Dept | 거래소 소속부 | str | **관리종목·스팩 판별** (아래) |
| Rank | 시총 순위 | int64 | — |

### Dept — 거래소가 직접 표시하는 상태값 (2017~2026 실측)

이름 추측보다 정확하다. `exclusions.py` 가 이 값으로 관리종목·스팩을 가른다.

| 값 | 행 수 | 비고 |
|---|---:|---|
| (결측) | 2,167,931 | KOSPI 는 대부분 비어 있다 |
| 중견기업부 / 우량기업부 / 벤처기업부 / 기술성장기업부 / 일반기업부 | 3,480,702 | 정상 |
| `SPAC(소속부없음)` | 146,118 | **제외 대상** |
| `관리종목(소속부없음)` | 124,033 | **제외 대상** |
| `투자주의환기종목(소속부없음)` | 86,874 | **제외 대상** |
| `외국기업(소속부없음)` | 41,879 | 미정 |
| 크라우드펀딩기업부 / 스타트업기업부 | 858 | KONEX |

> 결측이 두 종류다 — 진짜 `NaN` 과 **문자열 `"nan"`**. `isna()` 만으로는 안 걸러진다.

### 주의 — 종목코드 앞자리 0 (2000-05-29 이전)

marcap 은 6자리 코드 체계 도입(2000-05-29) 이전 코드를 **숫자로 저장**해 앞자리 0 이 날아가 있다
(삼성전자 `005930` → `"5930"`). 보정 없이 쓰면 코드 개편일에 1,460개 종목이 한꺼번에
상장폐지된 것처럼 보인다. `load_years()` 가 `zfill(6)` 으로 복원한다 (ADR-0003).
백테스트 구간(2017~)은 전부 6자리라 영향 없으나, **구간을 과거로 넓히면 반드시 걸린다.**

**검증 완료 (2026-07-20, `tests/test_marcap_loader.py`):**
- ✅ 망한 회사가 데이터에 남아 있는가 — 2017~ 구간 564개, 연도별 31~82개 고른 분포.
- ✅ `Marcap == Close × Stocks` — 604만 행 전부 일치, 불일치 0건. **단위는 원**
  (초안의 `/ 1e6`, 백만원 기재는 오류였다).
- ✅ 필수 컬럼 결측 0, `(Date, Code)` 중복 0, 연간 거래일 242~248일.
- ✅ 종목코드 6자리 정규화.

**아직 안 본 것:**
- `Amount == 0` 이 **4.24%** 있다 (거래정지·무거래). 처리 정책 미정 — 유동성 필터에서 어떻게 다룰지.
- 액면분할/병합 구간의 가격 점프 — 수정주가 여부 미확인. **백테스트 수익률에 직접 영향.**
- `Code` 재사용(상폐 후 코드 재부여) 여부 → 종목 식별 키 안정성.
- 데이터 갱신 지연 — 마지막 거래일이 실제보다 1~2 영업일 늦다. 백테스트엔 무관, 실시간 운용엔 부적합.

## 2. 종목 마스터 (point-in-time)

marcap의 날짜별 스냅샷에서 파생. 별도 테이블로 둘지, 패널에서 그때그때 뽑을지는 구현 시 결정.
필요 필드(안): `Code, Name, Market, 최초등장일, 최종등장일(≈상폐일 근사), 상장상태`.

## 저장 규약

- 원본은 커밋하지 않는다 (`data/` `.gitignore`). marcap은 clone, 나머지는 로컬 적재.
- 중간 산출물은 parquet. 파티션 키는 연도 또는 연·월 (구현 시 결정).
- 컬럼명은 소스 원본을 유지한다. (초안에 있던 "오타 컬럼 `ChagesRatio` 정규화" 규약은 **불필요** —
  실제 데이터는 `ChangesRatio` 로 정상이다.)
