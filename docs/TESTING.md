# 테스트 전략

백테스트 시스템에서 버그는 조용히 수익률을 부풀린다. 테스트의 1차 목표는 **방법론 가드레일을
코드로 강제**하는 것 — "돌아간다"가 아니라 "속이지 않는다"를 검증한다.

## 계층

1. **단위 테스트** — 순수 함수(신호 계산, 지표, 거래비용). 빠르고 결정론적. CI 기본.
2. **불변식 테스트** — 가드레일 계약. 아래 "반드시 잡아야 할 것" 참조.
3. **oracle 대조** (`@pytest.mark.slow`) — 단순 전략을 자체 엔진 vs backtesting.py로 돌려 P&L 일치 확인.
4. **데이터 무결성** (`@pytest.mark.slow`) — marcap 로드 후 상폐 종목 존재·정합성 (네트워크/대용량).

`slow` 마커가 붙은 것은 CI 기본 제외(`make test`), 로컬에서 `make test-all`로 실행.

## 반드시 잡아야 할 것 (불변식)

- **Look-ahead 없음.** 피처 계산에 "as-of 시점" 이후 행이 들어가면 실패. z-score 롤링 창에
  당일 포함 여부, 진입 시점 < 신호 시점(ADR-0001)을 명시 테스트.
- **Survivorship 없음.** 특정 과거 날짜의 universe에 이후 상폐된 종목이 포함되는지 확인.
  (오늘 살아있는 종목만 쓰면 편향 → 이게 잡혀야 함.)
- **3분할 격리.** Test 구간 데이터가 Train/Validate 경로로 새지 않는지. Test 접근 카운터.
- **거래비용 적용.** 비용 0 vs 0.5% 결과가 달라야 함 (비용이 실제로 붙는지).
- **회계 항등식.** 최종 자본 == 초기 자본 + Σ실현손익 − Σ비용. oracle과 ±허용오차 내 일치.
- **결정론.** 같은 입력 2회 실행 → 동일 출력(해시 비교).

## 원칙

- 임계값을 테스트에 하드코딩하지 않는다. 설정 주입값으로 검증(§0.1 placeholder).
- 지표는 손계산 가능한 tiny fixture로 검증 (예: 거래 3건짜리 원장의 WRL/Expectancy).
- 네트워크 의존 테스트는 반드시 `slow` 마커. 커밋 게이트를 네트워크에 걸지 않는다.

## 실행

```bash
make test        # 빠른 것만
make test-all    # slow 포함
pytest tests/layer4 -k lookahead -v   # 특정만
```
