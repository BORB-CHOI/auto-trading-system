# AI 자동 매매 시스템 — 프로젝트 지침서 v3.1

> **이 문서는 시스템 설계의 합의 사항·원칙·구조를 담은 살아있는 문서다.**
> 새 결정이 나오면 이 문서가 먼저 업데이트되어야 한다.
> 새 채팅 시작 시 이 문서를 먼저 첨부해서 컨텍스트로 사용한다.

---

## 0. 프로젝트 한 줄 정의

**15년차 직장인 스윙 트레이더의 매일 작업을 자동화하는 시스템 — 한국 시장에서 LLM과 정량 룰을 결합하여 실거래로 수익을 낸다.**

---

## 1. 사용자 (본인) 프로필

- **목표:** 실거래로 수익 (연구·공부 아님)
- **자본:** 1,000만원 이내. 단계적 투입 (100→300→1000만)
- **시장:** 한국 주식 (KOSPI/KOSDAQ). 미국은 1년+ 후 옵션
- **매매 스타일:** 스윙 (며칠~몇 주). 데이트레이딩·장기투자 ❌
- **상황:** 직장인 (월급 400~500만원, 직장가입자) 가정
- **프로그래밍:** 가능
- **종목 직관:** 없음 (시스템 트레이더에 적합)
- **메타 직관:** 있음 (기술적·시황·재료·재무·심리 결합 방향성)
- **LLM 활용 의지:** 강함, 특히 재료 해석 영역

### 1.1 직관 프로파일이 시스템에 의미하는 것

종목 직관 ❌ + 메타 직관 ⭕ = **시스템 빌더 적합형**.
- 본인이 시스템을 진짜로 따를 수 있음 (펀드매니저형은 못 따름)
- 시스템 결정에 본인 감정 개입 적음
- LLM이 종목 분석 역할 대신 수행해도 됨

### 1.2 시스템의 인간 프로토타입

15년차 살아남은 트레이더의 운영 방식 = 본인 시스템의 청사진:

```
매일 새벽 6시: 시황·관심테마 분석 (장 시작 전 시나리오)
9시 전: 예약매수·예약매도 주문 세팅
9~15:30: 회사 업무 (장중 안 봄)
저녁 9시: 오늘 회고 + 내일 시나리오 미리보기
주말: 큰 사건 없으면 안 함 ("대응 못 하는데 신경 안 씀")
```

본인 시스템은 이 워크플로우를 LLM으로 자동화하는 것. **시스템이 사람의 부재 시간을 메우는** 구조.

## 2. 시장 선택의 근거

### 2.1 한국 시장 선택 이유

| 요소 | 한국 | 미국 | 결정 |
|---|---|---|---|
| 매매차익 세금 | 0% (비과세) | 22% 분리과세 | 한국 ⭕ |
| 건보료 영향 | 없음 (배당 1,000만 미만) | 없음 | 동등 |
| 알파 가능성 | 상대적 높음 (Jacobs & Müller 2020) | 매우 어려움 | 한국 ⭕ |
| 외인/기관 일별 수급 | 무료 공개 | 분기 13F만 | 한국 ⭕ |
| 한국어 NLP 틈새 | 있음 | 영어 NLP 포화 | 한국 ⭕ |
| 자료/오픈소스 풍부함 | 부족 | 압도적 | 미국 ⭕ |
| 학습 곡선 | 가파름 | 완만 | 미국 ⭕ |
| 시드 1,000만 적합성 | 좋음 | 좋음 | 동등 |

**한국이 우위인 항목이 더 많음. 단, 자료 부족은 본인이 감수해야 할 비용.**

### 2.2 미국 시장 갈증 처리

- **자동매매 = 한국** (시스템적 결정)
- **장기 보유 = 미국 ETF/빅테크 적립** (개인 신념, 별도 계좌)

두 활동을 한 시스템에 섞지 않음.

## 3. 핵심 설계 원칙

### 3.1 4-layer + Layer 5 구조 (변경 금지)

```
[Layer 1] 데이터 수집
  ├─ 가격 / 거래량 / 호가
  ├─ 수급 (외인 / 기관 / 개인 일별)
  ├─ 뉴스 / 공시 / 유튜브 자막
  └─ 거시 지표

[Layer 2] 신호 생성  ← LLM이 들어가는 유일한 자리
  ├─ 텍스트 → 재료 분류 / sentiment / regime
  ├─ 종목별 narrative 해석
  └─ 출력: 종목별 conviction score + 근거 텍스트

[Layer 3] 포트폴리오 / 매매 전략  ← 정량 룰, 결정론적
  ├─ Universe filter (조건검색)
  ├─ 종목 선별 (대장주 판단)
  ├─ 포지션 사이징 + 비중 정책 (§6.8)
  ├─ 진입 / 청산 규칙
  ├─ 시나리오 플레이북 (§6.5)
  └─ 리스크 관리 (킬스위치 3종)

[Layer 4] 실행 + 백테스트
  ├─ 거래비용 / 슬리피지 / 시장충격 모델링
  ├─ 페이퍼 트레이딩
  ├─ KIS API 연동 실거래
  ├─ 예약 주문 안전장치 (§6.7)
  └─ 통계 검증

[Layer 5] AI 모니터링 (Phase 2~3)  ← 안전한 자기 개선
  ├─ 알파 부패 자동 감지
  ├─ 새 alpha 후보 가설 제안
  ├─ 프롬프트 개선 제안
  └─ 본인 승인 후 적용 (자동 적용 ❌)
```

### 3.2 LLM의 자리 (변경 금지)

**LLM이 하는 일 (Layer 2까지만):**
- 재료 다층 분석
- 시장 regime 판단
- 유튜브 시황 통합
- case retrieval용 임베딩
- 뉴스 trust_score 평가

**LLM이 절대 하지 않는 일:**
- BUY/SELL 직접 결정 ❌
- 포지션 크기 결정 ❌
- 손절 라인 결정 ❌
- 룰 자동 변경 ❌

LLM 출력 = 항상 정량 점수 + 태그. 자유 텍스트 매매 결정 금지.

### 3.3 LLM의 정성 분석 한계 (반드시 인지)

LLM이 약한 정성 영역:
- **메타 추론 (3~4층)** — 다층 사회적 추론
- **권력 동학** — 정치·외교·세력 의도 (적대적 정보 환경)
- **비공개 정보 활용** — 직관·암묵 노하우
- **반사실적 추론** — 일어나지 않은 일 시뮬레이션

LLM이 잘하는 정성 영역:
- 명시적 정보 추출 (재료 분류·강도·섹터 매핑)
- 직접 인과 추적
- 과거 사례 패턴 매칭

**활용 원칙:** LLM에 시나리오 *"근거"* 와 *"과거 case"* 는 시키되, 시나리오 *"확률"* 과 *"세력 진짜 의도"* 는 본인이 조정.

### 3.4 Universe Filter vs Signal 분리

```
KOSPI/KOSDAQ ~2,500종목
  ↓ Universe filter (결정론적 룰, DB 쿼리)
매일 30~80개 후보
  ↓ Signal (LLM + 수급 + 뉴스 결합)
오늘 진짜 살 3~5개
```

### 3.5 모델 학습은 단계적으로

- **Phase 1 (지금~6개월):** 학습 모델 0개. 범용 LLM API + 정량 룰만
- **Phase 2 (6~12개월):** case database 5만 건+ 후 임베딩·트리모델 검토
- **Phase 3 (1년+):** 부분 모듈에 한해 RL·파인튜닝 검토

**처음부터 파인튜닝 ❌.**

### 3.6 백테스트는 두 종류 모두

**(a) 표준 백테스트 (sanity check):**
- 모든 거래 합쳐 Sharpe, MDD, win rate
- 데이터 3분할 엄격 준수
- Test set은 **단 한 번만** 본다

**(b) 조건부 백테스트 (case-based, 본인 차별점):**
- 종목 × regime × 재료 조합별로 case database 구축
- "오늘 상황과 유사한 과거 case N건"의 통계 추출

(b)가 본인 시스템의 진짜 가치.

### 3.7 Regime-adaptive risk management

손절·익절 라인 고정 ❌. 시장 regime에 따라 동적 조정:

| Regime | 판단 기준 (예시) | 손절 | 익절 |
|---|---|---|---|
| 평시 | VIX < 20, 일일 변동 < 1% | -5% | +8% |
| 변동성 확대 | VIX 20~30 | -8% | +12% |
| 위기 | VIX > 30, 코로나·금리쇼크급 | -12% or 현금화 | — |

**불확실성 = 살아있는 바이러스 원칙:** 불확실성 미해소 상태에서는 위험이 시간 흐를수록 누적된다. 평시 룰을 그대로 적용하지 않고 비중을 사전에 축소.

### 3.8 예측 ❌, 대응 ⭕ (시스템 철학)

**원칙:** "시스템은 예측하지 않는다. 모든 시나리오에 사전 대응한다."

**예측 시도의 함정:**
- 매크로 충격(코로나, 트럼프 트윗, 전쟁)은 정의상 예측 불가
- 예측 가능했으면 시장이 이미 반영
- 예측 시스템 = overfitting의 본질
- LLM이 트럼프의 새벽 트윗을 예측? 트럼프 본인도 모름

**대응 시스템:**
- 모든 가능 시나리오에 사전 행동 정의
- 시나리오 발생 → 자동 실행
- 평가 기준: "맞췄다/틀렸다" ❌, "준비됐다/안됐다" ⭕

이게 본인 시스템과 학계 LLM 트레이딩 연구를 가르는 결정적 차이. 학계는 *"LLM이 다음날 가격 예측"* 에 집중. 본인은 *"모든 시나리오에 대응 정의"*.

## 4. 데이터 전략

### 4.1 데이터 3분할 (절대 원칙)

```
Train     2020-01 ~ 2023-12  (4년)  → 마음껏 실험
Validate  2024-01 ~ 2024-12  (1년)  → 골라낸 전략 검증
Test      2025-01 ~ 2025-12  (1년)  → 단 1회만
```

**Test 결과 보고 전략 고치면 그 순간 Test가 Train이 된다.**

장기적으로는 walk-forward 백테스트로 진화.

### 4.2 Universe Filter 조건 (현재)

본인이 검증해온 한국 스윙 조건:
- 시총 1,000억 ~ 3조
- 거래대금 100억 이상
- 1봉 전 종가 대비 0봉 전 종가 등락률 5% 이상
- 고가 등락률 15% 이상
- 최근 120봉 이내 52주 신고가 갱신

**학술적 정체:** Momentum effect + 52-week high anomaly 결합.

⚠️ regime에 따라 사라짐 (강세장·테마장 ⭕, 횡보·약세장 ❌).

### 4.3 Feature 후보

**Tier 1 — 정량, 가설 강함 (먼저 검증):**
- 외인 5일 누적 순매수 / 시총
- 기관 5일 누적 순매수 / 시총
- 거래대금 z-score (20일 평균 대비)
- 신고가 거리 (52주 고가 대비 %)
- 변동성 z-score (ATR 등)

**Tier 2 — LLM 생성, 본인 차별점:**
- 유튜브 패널 합의 점수
- 영상별 conviction 강도
- 뉴스 sentiment + 사건 분류 + trust_score
- 공시 sentiment + 공시 유형
- 시장 매크로 regime

**Tier 3 — 나중에:**
- 컨센서스 변화율
- 공매도 잔고
- 프로그램 매수 비중

**원칙:** Tier 1만으로 baseline → Tier 2 추가 시 알파 증가 측정.

### 4.4 재료 태깅 — Bottom-up 접근

미리 30개 카테고리 정의 ❌. 대신:
1. 처음에는 LLM에게 "한 줄 요약 + 핵심 키워드 3개"
2. 1~2개월 후 키워드 빈도 분석 → 자주 나오는 30~50개 표준 태그로 승격
3. 새 키워드는 계속 자유 (open vocabulary)

### 4.5 그래프 기반 case retrieval — 단계적

```
Phase 1: 태그 + sentence-transformers 임베딩 유사도 검색
Phase 2 (case DB 1만 건+): Neo4j 그래프 DB, 명시적 edge 추가
Phase 3 (검증 후): Graph Neural Network
```

**테마 종속성 그래프** (Phase 2 핵심): 종목 list가 아니라 테마 간 인과 그래프로 저장.

### 5.3 LLM
- **Claude / GPT API** — 텍스트 처리. 파인튜닝 ❌, 프롬프트 + RAG
- **sentence-transformers** — 임베딩 (case retrieval)

### 5.4 검증된 고수 유튜브 시황 (패널)

**구성 원칙:**
- 3~5명 패널
- 매일 장 시작 전(8시 이전) 업로드 우선
- 사전 시나리오형 우선 (회고형은 보조)
- 본인 매매 참고용으로만 (재배포 금지)

**LLM 추출 구조:**
```json
{
  "publish_date": "...",
  "main_variables": ["가장 중요한 변수 1~2개"],
  "scenarios": [{"name": "...", "probability": "...", 
                 "winners": [...], "losers": [...]}],
  "watchlist_by_theme": {"테마": ["대장주", "후속주", ...]},
  "earnings_calendar": {...},
  "warnings": [...]
}
```

**채널별 적중률 측정:**
- 영상 발행 후 1주/2주/4주 후 측정
- 시나리오 적중률, 종목 추천 수익률, 경고 정확도, 시점 정확도 분리
- 6개월 누적 → 채널별 강·약점 식별
- LLM이 영상 처리 시 **선택적 가중치** 적용

## 6. 매매 전략 (Layer 3)

### 6.1 핵심 측정 지표

승률 단독 평가 ❌. 다음 지표를 함께:

| 지표 | 의미 | 목표 |
|---|---|---|
| Win rate | 승률 | 50%+ |
| Avg win / Avg loss | 익손비 | 1.5+ |
| Profit factor | 총이익 / 총손실 | 1.5+ |
| **Max Drawdown (MDD)** | 최대 누적 손실 | regime별 차등 |
| Sharpe Ratio | 수익 / 변동성 | 1.0+ |
| Expectancy | 1회 거래 기대값 | **양수 필수** |

### 6.2 포지션 사이징 (개요)

- Kelly Criterion + conviction multiplier
- 단일 종목 최대 = 자본의 10% (§6.8)
- 동시 보유 = 5~10종목 (§6.8)

자세한 비중 정책은 §6.8 참고.

### 6.3 킬스위치 3종 (하드코딩, 런타임 변경 금지)

1. **종목별 손절** — regime별 차등 (§3.7)
2. **일일 손실 한도** — 자본의 -3%
3. **최대 누적 낙폭** — 자본의 -15% 도달 시 시스템 정지

세 개 다 코드 하드코딩. 환경변수 변경 ❌.

### 6.4 분할 매수 / 매도 (Strategic averaging)

**원칙:** 처음부터 분할 진입을 사전 계획. 풀 매수 ❌.

**진입 — 시드 분할 사전 정의:**
```
종목당 최대 비중 10% (자본 대비)

1차 진입: 5% (목표 비중의 50%)
2차 진입: -5% 하락 시 추가 3% (총 8%)
3차 진입: -10% 하락 시 추가 2% (총 10% = 한도 도달)

추가 매수 한도 도달 후:
  - 더 떨어져도 추가 매수 ❌
  - 시드 보호 우선
  - 다른 종목으로 분산
```

**중요:** 이건 *"감정적 물타기"* ❌, *"전략적 분할 진입"* ⭕. 매크로 충격(코로나·유가폭락 등)은 예측 불가하므로 처음부터 단계적 진입으로 대응.

**청산 — 사전 정의 비율:**
- 사전 정의 목표가 도달 시 일부 청산
- 트레일링 스탑은 나머지에 적용
- 단계별 비율은 §6.8

**호가:**
- 시장가 ❌, 지정가 + 호가창 슬리피지 모델링

### 6.5 시나리오 플레이북 (Scenario Playbook)

**원칙:** 시장이 움직이기 전에 의사결정이 끝나있어야 한다.

매주 일요일 (또는 큰 사건 트리거 시) 작성:

1. **다음 주 주요 이벤트 식별** — FOMC, 실적, 지정학, 정책
2. **메인 변수 1~2개 선정** — 시장 방향을 결정할 핵심 변수
3. **각 메인 변수별 3~4가지 시나리오** — 긍정/중립/부정/극단
4. **각 시나리오별 대응 사전 정의:**
   - 비중 조정 방향
   - 우선 공략 업종/종목
   - 진입 자리 (가격/시점)
   - 손절·익절 라인
5. **시나리오 트리거 신호 정의** (정량 임계값)

**LLM 활용:** 이벤트 식별 / 시나리오 생성 / 트리거 감지
**사람 역할:** 시나리오 검토·승인 / 너무 위험한 시나리오 제거 / 확률 가중치 조정 / 비중 한도 확인
**자동 실행:** 시나리오 트리거 발동 시 사전 정의 액션 실행

### 6.6 일일 운영 리듬 (Daily Cadence)

**원칙:** 직장인 스윙 트레이더의 일과를 시스템이 자동화.

```
[새벽 4~5시] 미국 마감 처리
  - 글로벌 뉴스·야간 선물·환율 분석
  - 시나리오 확률 갱신

[아침 6~8시] 장 시작 전 종합
  - 한국 뉴스·공시·고수 유튜브 자막 처리
  - 시나리오 플레이북 확정
  - 예약 주문 안 list 생성

[8:30~9:00] 본인 승인
  - 모바일 보고서 검토
  - 이상 종목 제거 또는 비중 조정
  - 승인 → KIS API 예약 주문 전송

[9:00~15:30] 장중 자동 운영
  - 예약 주문 자동 체결
  - LLM 호출 ❌ (긴급 알람만)
  - 본인 회사 업무

[15:30~17:00] 장 마감 후
  - 체결 결과 자동 정리
  - case DB 누적

[저녁 9~10시] 회고 + 다음날 미리보기
  - 일일 리포트 자동 생성
  - 다음날 시나리오 1차 안

[주말] 평시 휴식
  - 큰 사건 트리거 시에만 일요일 밤 분석
```

### 6.7 예약 주문 안전장치

**자동 취소 트리거:**
- 종목 거래 정지·관리종목 지정
- 갭 ±5% 이상 (개별 종목)
- 시장 전체 갭 ±2% 이상
- 주말 사이 큰 사건 발생
- KIS API 인증 오류·시스템 장애

**검증 룰:**
- 호가 폭발 시 슬리피지 한도 (지정가 ±0.5% 이내만)
- 부분 체결 정책 사전 정의
- 거래 정지·일시 중단 시 무효화 후 재평가

### 6.8 비중 정책 (Position & Capital Management)

**원칙:** 시드 100% 활용 강제 ❌. 분산이 집중보다 우선.

#### 현재 확정 룰 (Phase 1 적용)

**종목당 한도:**
- 한 종목당 자본의 **최대 10%**
- 동시 보유 **5~10종목** 가능
- 시드 100% 활용 강제 ❌ (현금 보유 정상)

**분할 진입 (§6.4 참고):**
- 사전 계획 분할 (1차 5% / 2차 3% / 3차 2%)
- 종목당 한도 10% 도달 시 추가 매수 ❌
- 매크로 충격은 예측 ❌, 분할 진입으로 대응 ⭕

**청산 룰:**
- 사전 정의 목표가 도달 시 일부 청산 (비율 TBD, 백테스트로 결정)
- 트레일링 스탑은 나머지에 적용
- regime별 손절·익절 라인 (§3.7)

**진입 정책:**
- 손절 후 시간 기반 휴식 ❌ (시장 조건이 트리거)
- 시장 조건 충족 시 즉시 재진입 가능
- 시장 신호가 결정, 본인 평단가는 결정 요소 ❌

**비중 정상화 룰 (Position normalization):**
- 분할 진입 후 한도 초과 시 → 가격이 평단 위로 올라온 즉시 한도까지 청산
- "본전 회복" 자체는 트리거 ❌, "비중 한도" 가 트리거 ⭕
- 시장은 본인 평단가를 모르기 때문

**포지션 리프레시 (Position refresh):**
- 분할 진입 후 일정 기간(예: 30일) 이내 평단 도달 시
- 강제 재평가: *"지금 시점에 처음 진입한다고 가정, 여전히 매수할 만한가?"*
- 아니면 청산

#### 미결정 — Phase 1 진행하며 데이터로 결정

- [ ] regime별 전체 시장 노출 비중 (Market exposure %)
- [ ] 신규 vs 추가매수 시드 분배 비율
- [ ] 시나리오 확률 → 사이징 매핑 공식
- [ ] Conviction multiplier 계산식
- [ ] 단계별 부분 청산 비율 (5%/10%/20% 도달 시)
- [ ] 동시 보유 종목 수 상한 (5~10 중 어디)
- [ ] 분할 진입 횟수 상한 (현재 3회)
- [ ] 하락장 비중 정책 (경험 축적 후)
- [ ] Crisis용 별도 시드 풀 운영 여부

#### 본인 룰의 분해 (자기 점검 결과)

| 룰 | 합리적 | 심리 | 처리 |
|---|---|---|---|
| 종목당 최대 10% | 100% | 0% | 그대로 적용 |
| 목표가 도달 시 일부 청산 | 80% | 20% | 행동은 유지, 임계값은 백테스트로 |
| 본전 회복 시 절반 청산 | 50% | 50% | "비중 한도" + "포지션 리프레시"로 분해 |

본인의 *"본전 청산"* 룰의 심리 부분은 시스템에서 객관적 트리거로 대체.

## 7. 5년 생존 원칙

### 7.1 알파 부패 인정

**학술적 사실:**
- 미국 시장: 발표된 anomaly 평균 −58% 부패. 거래비용 후 −93%
- 한국·일본·유럽 등 비미국: 부패 통계적으로 덜 명확 (Jacobs & Müller 2020)
- 부패 속도 가속화 (1980s 수년 → 2020s 수시간)

**시사점:**
- 한 알파에 의존 ❌
- 정적 시스템 ❌
- 6개월~1년마다 재검증·재학습 필수
- 본인만의 데이터(한국 유튜브 시황 등) 활용 알파는 더 오래 살아남음

### 7.2 다층 알파 구조 (Slot 시스템)

```
Slot 1: universe filter + 단순 momentum
Slot 2: 외인·기관 수급 신호
Slot 3: 재료 해석 (LLM)
Slot 4: 유튜브 시황 합의 신호
Slot 5+ : (반기 1개 페이스) 새 alpha
```

각 slot 수익 기여도 매월 측정. 부패하는 slot은 비중 ↓.

### 7.3 LLM 자원 우선순위 (부패 느린 자리에 집중)

```
1순위 ⭐⭐⭐: 재료 다층 분석
2순위 ⭐⭐⭐: 시장 regime 판단 (하루 1회)
3순위 ⭐⭐: 유튜브 시황 패널 통합
4순위 ⭐: 종목별 sentiment (다들 함, 부패 빠름)
순위 밖 ❌: 차트 패턴 인식
```

본인 직관 *"재료 파악이 답이다"* 가 1순위와 정확히 일치.

### 7.4 손실 관리 우선

> *"인내심 없는 사람의 돈이 인내심 있는 사람에게로 이동한다."*
> *"고수는 변동성을 이용. 초보는 변동성에 이용당한다."*

알파보다 리스크 관리가 우선. § 6.3 킬스위치 + § 6.5 시나리오 플레이북 + § 6.8 비중 정책에 박혀 있음.

### 7.5 적응 메커니즘 (Layer 5, Phase 2~3)

```
주간: slot 수익 기여도 측정, 이상 패턴 감지
월간: 알파 부패 감지(IC 추세), 새 alpha 후보 가설 3개
분기: 시스템 진단, slot 비중 조정 제안, regime 변화 분석
```

**모든 제안은 본인 검토 후 수동 적용.** Test set 검증 + 페이퍼 트레이딩 2주 → 라이브.

### 7.6 새 alpha 발굴 워크플로우

```
[1단계] 시스템: 후보 자동 발굴 (매주)
  ±3σ 이탈 통계 추출 → 의미 있는 5~10개 list

[2단계] LLM: 가설 생성 (자동)
  각 후보에 대해 가능한 가설 5개 + 검증 방법
  
[3단계] 사람: 검증·승인 (수동)
  - 말 되는 가설 1~2개 선정
  - case DB에서 과거 유사 case 검색
  - Train set 백테스트 → Validate 확인
  - 통과 → 새 alpha slot으로 추가

빈도:
  후보 발굴: 매주 5~10개
  사람 검증 시도: 매월 1~2개
  Train/Validate 통과: 분기 1개
  실거래 도입: 반기 1개
```

## 8. 자기 개선의 한계 (반드시 인지)

### 8.1 트레이딩에서 완전 자율 자기 개선이 어려운 4가지 이유

1. **신호 노이즈** — 매매 결과에 운/실력 섞임 (low SNR)
2. **데이터 부족** — 일봉 30년 = 7,500개. 모두 다른 regime
3. **적대적 환경** — 시장이 본인 패턴을 노리고 진화
4. **실패 비용 비대칭** — 큰 손실 한 번이 영구적

### 8.2 안전한 자기 개선 가능 영역

✅ 정량 hyperparameter (손절 라인 등 숫자)
✅ 프롬프트 개선 (A/B 테스트)
✅ alpha slot 비중 조정
✅ 재료 분류 정확도 (라벨 명확)
✅ 새 alpha 후보 발굴 (가설 생성까지만, 승격은 수동)

❌ 새 alpha 자동 도입 (창의적·적대적)
❌ 시장 regime 자체 판단 (새 regime은 항상 처음)
❌ 세력 의도 추론 (적대적 환경 핵심)
❌ 다층 메타 추론 (정치 동학)

### 8.3 원칙: AI는 분석가, 사람은 결정자

```
AI = 측정·감지·제안·가설
사람 = 검증·승인·책임
시스템 = 결정론적 룰 실행
```

## 9. 실거래 개입 정책 + 트리거 시스템

### 9.1 본인 개입의 통계적 함정

**Barber & Odean (2000)** "Trading is Hazardous to Your Wealth":
- 거래 빈도 상위 20%: 연 11.4% 수익
- 거래 빈도 하위 20%: 연 18.5% 수익
- **차이 7%p = 본인 개입 비용**

처분 효과 / Loss aversion / 최근성 편향 / 확증 편향이 본인을 갉아먹음.

### 9.2 그러나 완전 자동화도 위험

- 시스템이 못 보는 영역 (대주주 횡령 등)
- 모델 고장 (API 오류·데이터 누락)
- 백테스트에 없던 regime 전환점

→ 사람이 빠르게 판단할 수 있는 자리는 분명 존재.

### 9.3 Multi-Tier 트리거 시스템

**원칙:** 시장 인과 순서 따라 우선순위 설정. 가격은 **결과**, 수급/뉴스가 **원인**.

```
사건 발생
  ↓
세력/기관 정보 입수 (T-N일)
  ↓
조용한 매집 → 수급 데이터 변화 (Tier 1)
  ↓
뉴스/SNS 보도 (Tier 2)
  ↓
사람들이 읽고 반응
  ↓
가격 움직임 (Tier 3) ← 결과
```

#### Tier 1: 수급 트리거 (Pre-announcement)

**가장 빠른 정보 — 한국 시장의 핵심 알파 자리.**

```
시그널:
  - 외인 5일 누적 z-score
  - 기관 5일 누적 z-score
  - 외인+기관 동시 매수 (강도 ↑)
  - 개인 매수 + 외인 매도 (역신호)

의미: "세력이 뭔가 알고 사고 있다"
한국 시장의 일별 수급 공개가 미국에 없는 우위.
세력 정보 우위의 그림자를 추적.
```

#### Tier 2: 뉴스/SNS 트리거 (정보 공식화)

**메인 트리거 (사람들이 보고 반응하는 시점).**

**채널:**
```
초고속 (분 단위):
  - 로이터, 블룸버그, 연합인포맥스 (브레이킹)
  - DART 실시간 공시
  - X(트위터) 사전 정의 계정

중간 (시간 단위):
  - 매일경제, 한경, 조선비즈
  - 정부 보도자료
  - 외신 (NYT, WSJ, FT)

느림 (일 단위):
  - 분석 기사, 사설
  - 증권사 리포트
```

**LLM trust_score 평가:**

```python
# 매체 신뢰도 등급
Tier S: 통신사·공식 공시 → 가중치 1.0
Tier A: 주요 일간지 → 가중치 0.8
Tier B: 인터넷 언론·블로그 → 가중치 0.5
Tier C: 출처 불명·카더라 → 가중치 0.2

# 단독 vs 복수 보도
단독 보도 (1개 매체) → 0.3~0.5 (오보 위험 ↑)
복수 보도 (2+ 매체) → 0.8~1.0
공식 발표 → 1.0

# 시간 경과 갱신
T+0: 초기 신뢰도 (60% 가정)
T+30분 다른 매체 인용 → 80%
T+1시간 당사자 코멘트 → 90%
T+1일 정정 없음 → 95%
정정 보도 발생 → 0.0 + 행동 되돌리기

# 작전 패턴 감지 (가중치 ↓)
- 같은 종목 단기 호재 도배
- 동일 매체 반복 추천
- 보도 후 즉시 거래량 폭증 (사전 정보 의심)
- 익명 소스만, 확인 안 됨

# 종합 trust_score = 위 차원 곱
```

**LLM 처리 출력:**

```json
{
  "headline": "...",
  "source": "매일경제",
  "source_tier": "A",
  "is_exclusive": false,
  "corroborating_sources": 3,
  "official_confirmed": true,
  "trust_score": 0.85,
  "content_classification": {
    "type": "신규수주",
    "magnitude": "large",
    "direct_indirect": "direct",
    "affected_tickers": [...]
  },
  "suspicious_patterns": []
}
```

**트위터/X 계정:**
- 사전 정의 계정 list 모니터링 (회사 IR, CEO, 금융 전문 기자)
- 영향력 가중치 (팔로워 + 과거 적중률)
- 가짜·해킹 의심 → 다른 채널 검증 후 행동

#### Tier 3: 가격 트리거 (반응 검증)

```
시그널:
  - 거래량 z-score
  - 가격 ±N% 이동
  - 호가창 변화
  
역할: 검증·확정용 (단독 메인 ❌)
의미: "시장이 반응하고 있다" — 이미 늦은 신호일 가능성
```

#### Tier 4: 광범위 시장 충격 트리거 (예외)

매크로 충격은 수급으로 못 잡음. 별도 처리:

```
트리거:
  - KOSPI 1일 변동 -3% 이상
  - 야간 선물 ±2% 이상
  - VIX 30 돌파
  - USD/KRW 1일 1.5%+
  - 유가 1일 5%+
  - 미국 지수 -2% 이상

자동 액션:
  - 신규 진입 자동 중단
  - 예약 주문 자동 취소
  - 기존 포지션 즉시 청산 ❌ (오히려 매수 기회 가능성)
  - 본인에게 알람
  - 다음 장 시작 시 시나리오 재평가
```

#### 조합 룰

```
Tier 1 + Tier 2 동시 발동 + trust_score 높음
  → 가장 강한 알파 신호
  → 사전 정의 행동 실행

Tier 1만 단독
  → 잠재 신호, 관찰 (수급 변화 누가 만들었나?)

Tier 2만 단독 + trust_score 높음
  → 약한 신호, 신중 진입

Tier 2만 + trust_score 낮음 (가짜 의심)
  → 무시

Tier 3만 단독
  → 검증 필요, 행동 보류 ("왜 움직였지?")

Tier 4 단독
  → Crisis 모드 (예측 불가 충격)
  → 신규 진입 중단 + 시나리오 재평가
```

#### 커뮤니티 트리거 (Phase 2 이후, 방어용만)

**Phase 1에서는 ❌.**

Phase 2 도입 시:
- 매수 신호 ❌, **매수 금지/방어 신호로만**
- 활동량 z-score (내용 ❌)
- 광기 지표 (역신호) — 천장 의심
- 작전 의심 감지 — 종목 자동 universe 제외

### 9.4 모드별 개입 정책

```
[Mode A: 정상 운영]
  본인 개입 ❌
  시스템 룰 그대로 실행
  본인은 보고서만 봄
  
[Mode B: 비상 모드]
  본인 개입 ⭕ (제한적)
  발동 조건: Tier 4 트리거 / 시스템 알람 / 회사 정책 변경
  권한:
    ✅ 전체 포지션 청산
    ✅ 신규 진입 중단
    ❌ 개별 종목 임의 매수
    ❌ 손절 라인 풀기
  
[Mode C: 학습 모드]
  본인이 시스템 출력 보고 가설 생성
  실거래 영향 ❌
  case DB·프롬프트 개선용
```

**핵심 원칙: 개입의 방향이 비대칭이어야 한다.**
- ✅ 허용: 포지션 줄이기, 청산, 진입 중단 (보수적)
- ❌ 금지: 포지션 늘리기, 임의 매수, 손절 풀기 (공격적)

### 9.5 대응 불가능 원칙

> **"대응 불가능한 시간대에 분석하지 않는다."**

- 장중 LLM 호출 ❌
- 주말 평시 분석 ❌
- 야간 1회만 (미국 마감 후)
- 큰 사건 트리거 시에만 예외 분석

### 9.6 단계별 개입 수준

```
Phase A (100만원, 0~2주): 완전 자동, 개입 0
Phase B (300만원, 2~6주): Mode A 위주, 비상 트리거만
Phase C (1000만원, 6주+): Mode A + B + C
```

**시드가 작을수록 개입 ❌, 클수록 개입 허용 (제한적).**

## 10. 실패 방지 원칙

### 10.1 절대 하지 말 것

- ❌ Test set 보고 전략 수정
- ❌ 백테스트 통과 즉시 풀시드 투입
- ❌ 시장 충격 큰 종목에 시장가 진입
- ❌ 킬스위치 우회 / 무시
- ❌ "이번엔 다르다" 직관에 따른 수동 개입
- ❌ LLM 좋다고 한 종목을 검증 없이 매수
- ❌ 단일 유튜브 채널 의존
- ❌ 거래비용·슬리피지 무시한 백테스트 신뢰
- ❌ AI에게 룰 자동 변경 권한 부여
- ❌ 단일 alpha에 100% 의존
- ❌ Mode B에서 포지션 늘리기·손절 풀기
- ❌ 단독 보도 trust_score 무시

### 10.2 항상 의심해야 할 것

- *"내 전략은 무엇 때문에 작동하지 않을까?"*
- *"발견한 알파는 언제 부패할까?"*
- *"이게 우연일 확률은?"*
- *"지금 시장 regime이 백테스트 기간과 다른가?"*
- *"내가 개입하고 싶은 이 충동은 시스템 결함 때문인가, 내 심리 때문인가?"*
- *"이 신호는 시장 신호인가, 내 평단가인가?"*

### 10.3 자본 투입 단계

```
Phase A: 100만원   → 2주+ 안정 → 통과
Phase B: 300만원   → 4주+ 안정 → 통과
Phase C: 1,000만원 → 본격 운영
```

각 단계 안정성 = "백테스트 통계와 실거래 결과가 ±20% 이내 일치"

## 14. 본인 시스템의 정체성 (한 문단 요약)

> 본인은 종목 직관 없이 메타 직관을 가진 시스템 빌더다. 한국 시장의 비효율(외인/기관 일별 수급 공개, 한국어 NLP 틈새, 알파 부패 느림)을 노려 **15년차 직장인 스윙 트레이더의 매일 작업을 자동화하는 시스템**을 만든다. 새벽에 시나리오 플레이북, 장 시작 전 예약 주문, 장중 무개입, 저녁 회고 — 이 일과를 LLM과 정량 룰로 코드화한다. **시스템은 예측하지 않고 모든 시나리오에 사전 대응한다.** LLM은 재료 해석·시황 판단·뉴스 trust_score에만 쓰고 매매 결정은 정량 룰. Multi-tier 트리거(수급→뉴스→가격)로 시장 인과 순서대로 신호를 잡고, 매크로 충격은 분할 진입으로 대응한다. 표준 백테스트로 sanity check, 조건부 백테스트(case DB)로 실제 결정. 단일 알파 ❌, 다층 slot으로 분산. AI는 분석가·사람은 결정자. 실거래 개입은 모드별로 비대칭(보수 방향만 허용). 1,000만원 시드를 단계적으로 투입하며 검증.
