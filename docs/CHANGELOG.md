# CHANGELOG

각 버전의 핵심 변경만 한눈에 볼 수 있도록 요약한다.

원칙:
- 새 버전은 **위쪽**에 추가 (역순)
- 각 버전마다 **추가 / 변경 / 정정 / 제거** 로 분류
- 한 줄당 §번호와 한 줄 요약
- 상세 내용은 본 지침서에서 확인

## v3.1 (현재) — 2026-04-26

**테마: 시장 인과 순서 반영 + 비중 정책 통합 + 예측 ❌ 대응 ⭕ 철학 명문화**

### 추가
- **§3.8 신규** — 예측 ❌ 대응 ⭕ 시스템 철학. 매크로 충격은 정의상 예측 불가, 모든 시나리오에 사전 대응 정의로 다룬다.
- **§6.8 신규** — 비중 정책 통합 섹션. 종목당 10% / 동시 5~10종목 / 분할 진입 사전 계획 / 비중 정상화 / 포지션 리프레시. 본인 룰의 합리/심리 분해 결과 포함.
- **§9.3 재작성** — Multi-Tier 트리거 시스템 (Tier 1 수급 → Tier 2 뉴스/SNS → Tier 3 가격 → Tier 4 매크로). 시장 인과 순서 반영.
- **뉴스 trust_score 시스템** (§9.3 내) — 매체 신뢰도 등급, 단독 vs 복수 보도, 시간 경과 동적 갱신, 작전 패턴 감지.
- **§17 학술 근거 2개 추가** — Cohen et al. (2012, 정보 우위 → 가격), Shefrin & Statman (1985, Disposition Effect)

### 변경
- **§6.4 강화** — 분할 매수를 *"전략적 사전 계획"* 으로 명시. 1차 5% / 2차 3% / 3차 2% 단계 명시. *"감정적 물타기"* 와 구분.
- **§6.2 정정** — 동시 보유 *"3~5종목"* → *"5~10종목"* (종목당 10% 한도와 일치)
- **§9 구조 변경** — *"실거래 개입 정책"* 에서 *"실거래 개입 정책 + 트리거 시스템"* 으로 통합. 트리거가 개입 모드와 직결됨.
- **§14 정체성 갱신** — 예측 ❌ 대응 ⭕ + Multi-tier 트리거 명시
- **§10.1 추가 항목** — *"Mode B에서 포지션 늘리기·손절 풀기 ❌"*, *"단독 보도 trust_score 무시 ❌"*

### 정정
- **물타기 개념 정정** — 이전 v3에서 *"잘못된 진입의 결과"* 로 깔린 것을 *"전략적 사전 계획"* 으로 명확화. 매크로 충격이 예측 불가하기 때문에 분할 진입이 필수 전략.

### 영향
- 1주차 데이터 파이프라인 설계 시 **수급 데이터를 가장 우선** 처리해야 함 (Tier 1 트리거).
- LLM 신호 모듈에 **trust_score 출력**을 명시적으로 포함해야 함.
- 백테스트 엔진에 **분할 진입 시뮬레이션** 기능 필요.

## v1 — 2026-04-26 (초안)

**테마: 4-layer 구조 정의 + 데이터 3분할 + Regime-adaptive**

### 초기 정의
- **§3 4-layer 분리** (Layer 1 데이터 / Layer 2 신호(LLM) / Layer 3 매매(정량) / Layer 4 실행)
- **§3 LLM의 자리** — Layer 2까지만, 매매 결정 ❌
- **§3 Universe Filter vs Signal 분리**
- **§4 데이터 3분할 절대 원칙** (Train/Validate/Test, Test는 단 1회)
- **§3 백테스트 두 종류** — 표준 + 조건부(case-based)
- **§3 Regime-adaptive risk** — 손절 라인 동적
- **§4 Universe Filter 조건** — 본인의 시총·거래대금·등락률·신고가 조건
- **§5 Feature Tier 분류** — Tier 1 정량 / Tier 2 LLM / Tier 3 나중에
- **§6 매매 전략** — Kelly + conviction, 킬스위치 3종, 분할 매수
- **§9 자본 투입 단계** — Phase A 100만 → B 300만 → C 1000만

## 버전 호환성

각 버전은 이전 버전의 **상위 호환**이다. v3.1을 사용하면 v1, v2, v3의 모든 원칙을 포함한다.
