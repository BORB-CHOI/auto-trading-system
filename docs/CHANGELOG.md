## v3.4 (현재) — 2026-04-26

**테마: 영상 2개 통찰 흡수 — Conviction 이중 게이트 + 청산 4-카테고리 + 측정 지표 정밀화**

(15년차 트레이더의 *"기법 만들기"* + *"수익 확률 올리는 방법"* 두 영상 통찰 + 본인 본절 룰 분해 작업 통합)

### 추가

**구조 / 메타 원칙:**
- **§0.1 신규** — 숫자 표기 원칙 (브레인스토밍 단계). 모든 임계값·가중치는 placeholder, 데이터로 교체. *"기준 흔들기"* vs *"placeholder 교체"* 구분.
- **§3.11 신규** — 룰 분해 원칙 (Rule Decomposition Meta-principle). 모든 룰은 표면 트리거 ❌, 5종 신호 카테고리(Position / Market / Hypothesis / Capital / Time) 조합으로 정의. 본절 룰 분해 작업의 일반화.

**Conviction 시스템 (영상 2 통찰):**
- **§6.2 대폭 강화** — Conviction 이중 게이트. Conjunctive(각 신호 최저 통과) AND Compensatory(가중 평균 통과). 5개 신호 (chart / supply / material / macro / panel) 점수화. Conviction → 비중 매핑 (75-80 / 80-90 / 90-95 / 95+). Kelly fraction × 0.5 통합.
- **§6.8 강화** — Conviction 점수 → 비중 매핑 명시. 95+ 시 일시 비중 한도 12% 허용. *"비중 배팅이 계좌 펌핑 핵심"* (영상) 구현.
- **§14 정체성 갱신** — *"1등 전략 ❌, 상위 10% 전략 ⭕"* 명문화. Conjunctive + Compensatory 이중 게이트 정체성 한 줄 추가.

**측정 지표 정밀화 (영상 1 + 영상 2):**
- **§4.6 강화** — Win Rate Lift (영상 직관 *"50대50에서 시작"*) + Information Coefficient (학술 표준 Spearman ρ) 분리 정의. 둘 다 측정해서 비교. WRL은 직관, IC는 엄밀. 둘이 모순될 때 원인 추적. IC 가산성 검증 (다중 신호 결합 시 중복 감지).
- **§4.6 표본 크기 임계값** — N < 30 가설 / 30~100 약한 / 100~500 중간 / 500+ 강한 / 1,000+ 매우 강한. Phase 1 4년 데이터에 표본 100+ 안 되는 anchor는 slot 채택 보류.
- **§6.1 재구성** — 측정 지표를 Layer 1 (단일 신호: WR / WRL / IC) + Layer 2 (전체 시스템: Sharpe / MDD / Expectancy 등) + Layer 3 (청산 카테고리별) 로 계층화.

**청산 4-카테고리 (영상 1 + 본절 분해):**
- **§6.4 대폭 강화** — 청산을 익절 / 본절 / 손절 / 포지션 리프레시 4-카테고리로 분리. 각 카테고리별 트리거를 §3.11 신호 분해로 명시. 백테스트 통계 카테고리별 독립 측정.
- **§6.4 거래비용 다단계 검증** — 수준 0 (0%, 사용 ❌) / 수준 1 (0.3%, 수수료만) / 수준 2 (0.5%, default) / 수준 3 (0.8%+, stress test). 영상의 *"수수료 때문에 익절 못 한 종목"* 통찰 반영.

**기준 vs 세부 분리:**
- **§4.1 강화** — *"anchor (기준)는 절대 흔들지 않고 세부 수치만 조정"* 영상 메소드론 명문화.

### 변경

- **§14 정체성 한 문단** — Conjunctive + Compensatory 이중 게이트, 1등 ❌ / 상위 10% ⭕, 룰 분해 §3.11, WRL + IC + Expectancy 셋 다 측정, 표본 N < 100 보류, 거래비용 0.5% default, 청산 4-카테고리 추가

### 정정

- **§6.1 측정 지표** — *"승률 단독 평가 ❌"* 의 의미를 *"WR + WRL + IC + Expectancy + 표본 크기 충족 시"* 로 구체화

핵심 통합:
- 영상 1 *"기법 만들기"* (수익 안 나는 → 통계 + 강화·조율) + 본인 본절 분해 → §3.11 룰 분해 원칙
- 영상 2 *"수익 확률 올리는 방법"* (다요소 점수화 + 비중 배팅) → §6.2 Conviction 이중 게이트
- 영상 1 표본 크기 + 거래비용 강조 → §4.6 + §6.4 정량 게이트
- 양쪽 영상 *"50대50 시작"* → §4.6 Win Rate Lift 정량화

### 영향

- **§10.1 추가 항목** — anchor 없는 fuzzy 통계 / Feature 검증 안 한 채택 / regime 무시 단일 섹터 집중 모두 금지.

- **§15 사용법 추가** — anchor 헷갈릴 때 §3.10, feature 검증 §4.6, 섹터 §4.7, slot 비중 §7.2.

### 영향

- 1주차 데이터 파이프라인 설계 시 **수급 데이터를 가장 우선** 처리해야 함 (Tier 1 트리거).
- LLM 신호 모듈에 **trust_score 출력**을 명시적으로 포함해야 함.
- 백테스트 엔진에 **분할 진입 시뮬레이션** 기능 필요.
