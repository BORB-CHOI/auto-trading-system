### 추가
- **§3.12 신규** — 선반영 + 기대감 해소 메커니즘 (Anticipation Discharge). §3.9 surprise 와 별개의 독립 메커니즘으로 명문화. 양방향 작동 (호재 선반영 → 기대감 해소로 하락 / 악재 선반영 → 불안감 해소로 상승). FOMC, CES 같은 큰 이벤트에서의 *"그래서 쏘 왓?"* 현상 정량화.
- **§3.9 + §3.12 결합 매트릭스** — surprise 부호 × 선반영 강도 = 가격 반응 예측 매트릭스. 시나리오 플레이북의 핵심.
- **§4.3 강화** — Tier 2 feature에 anticipation_buildup 점수 / discharge_risk_score 추가.
- **§4.4 강화** — 재료 태깅 메타데이터에 anticipation 차원 (buildup_period_days, cumulative_price_move_pre_event, anticipation_intensity, discharge_risk_score) 추가.
- **§6.5 강화** — 시나리오 플레이북에 선반영 강도 측정 단계 추가 (3단계). 동일 surprise라도 선반영 강도가 다르면 반응이 다르다는 매트릭스 명시.
- **§9.3 강화** — Tier 2 LLM 처리 출력에 anticipation_dimension 필드 추가. 최종 신호 강도 공식: `trust × |surprise| × freshness × (1 - discharge_risk_score)`.
- **§17 학술 근거 4개 추가** — Brown & Warner (1985, event study pre-event drift), Frazzini & Lamont (2007, earnings announcement premium), Savor & Wilson (2013, FOMC risk premium), Hong/Lim/Stein (2000, gradual information diffusion).
- **§10.1 추가 항목** — 선반영 무시 / "기대 적중인데 왜 떨어졌지?" 사후 합리화 / "Buy the rumor, sell the news" 무차별 적용 / 갑작스런 충격에 §3.12 적용 모두 금지.

- **§13 TBD 신규 카테고리** — 선반영 메커니즘 미결정 항목들 (강도 분류 임계값, 측정 기간, α/β 가중치, 옵션 IV 데이터 소스 등).

## v3.4 — 2026-04-26

**테마: Conviction 이중 게이트 + 청산 4-카테고리 + Win Rate Lift × IC + 거래비용 다단계 + 룰 분해 메타원칙**

### 추가
- **§3.11 신규** — 룰 분해 원칙 (Rule Decomposition Meta-principle). 모든 매매 룰은 표면 트리거가 아니라 신호 조합으로 정의. 5종 신호 카테고리 (Position / Market / Hypothesis / Capital / Time).
- **§6.2 재작성** — 포지션 사이징 + Conviction 이중 게이트. 신호별 점수 (s_chart, s_supply, s_material, s_macro, s_panel) → Conjunctive 게이트 (각 신호 최저 통과) + Compensatory 게이트 (가중 평균 통과). Tversky (1972) 다속성 의사결정 학술 근거.
- **§6.4 재작성** — 청산 4-카테고리 (익절/본절/손절/포지션 리프레시) + 거래비용 다단계 검증 (수준 0~3, default 0.5%).
- **§6.1 강화** — 측정 지표 3-Layer 구조 (단일 신호 / 전체 시스템 / 청산 카테고리별). Win Rate Lift + IC 학술 표준 분리.
- **§4.6 재작성** — Feature 검증 룰을 Win Rate Lift + IC 두 측정으로 정형화. 표본 크기 신뢰성 임계값 (N < 100 slot 채택 보류).
- **§13 TBD 신규 카테고리들** — Conviction 시스템 / 청산 4-카테고리 / WRL+IC 임계값 / 거래비용 / 룰 분해.
- **§17 학술 근거 추가** — Tversky (1972, Elimination by Aspects).

### 변경
- **§14 정체성 갱신** — Conviction 이중 게이트 + 4-카테고리 청산 + 룰 분해 + 표본/거래비용 게이트 명시.
- **§0.1 추가** — 숫자 변경 가이드 (§0.1) 안내.
- **§7.2 Slot 측정** — Win Rate Lift + IC 명시 (이전 막연한 *"승률"* 대신)

### 정정
- **IC 정의 정정** — 이전 비공식 표현 *"승률 - 50%"* 를 *"Win Rate Lift"* 로 명명. 학술 IC는 Spearman 상관계수로 별도 정의. 두 측정을 명확히 분리.
- **§6.1 측정 지표** — *"승률 단독 평가 ❌"* 의 의미를 *"WR + WRL + IC + Expectancy + 표본 크기 충족 시"* 로 구체화

### 의미

이번 v3.4의 본질은 **본인이 1년 매매로 직접 학습할 휴리스틱을 영상의 고수로부터 압축 흡수 + LLM과 데이터로 검증** 하는 구조의 명문화. 영상의 직관(50대50, 다요소 점수화, 1등 vs 상위 10%, 비중 배팅)을 학술 표준(Tversky 다속성 의사결정, Spearman 상관, IC, expectancy)과 매핑하여 본인 시스템에 박음.

핵심 통합:
- 영상 1 *"기법 만들기"* (수익 안 나는 → 통계 + 강화·조율) + 본인 본절 분해 → §3.11 룰 분해 원칙
- 영상 2 *"수익 확률 올리는 방법"* (다요소 점수화 + 비중 배팅) → §6.2 Conviction 이중 게이트
- 영상 1 표본 크기 + 거래비용 강조 → §4.6 + §6.4 정량 게이트
- 양쪽 영상 *"50대50 시작"* → §4.6 Win Rate Lift 정량화

### 영향

브레인스토밍 단계 산출물. 코드 작성 시작 시 영향:
- 1주차 데이터 파이프라인에 conviction 5개 신호 컬럼 + 청산 카테고리 분류 + 거래비용 단계별 시뮬레이션 필요
- 백테스트 엔진은 카테고리별 통계 분리 + WRL + IC 둘 다 출력 + 표본 크기 자동 게이트 필요
- LLM 프롬프트에 신호별 점수 (s_chart / s_supply / s_material / s_macro / s_panel) 출력 강제
- case DB 스키마에 청산 카테고리 + conviction 점수 + 신호별 분해 컬럼 추가

- **§9.3 재작성** — Multi-Tier 트리거 시스템 (Tier 1 수급 → Tier 2 뉴스/SNS → Tier 3 가격 → Tier 4 매크로). 시장 인과 순서 반영.
- **뉴스 trust_score 시스템** (§9.3 내) — 매체 신뢰도 등급, 단독 vs 복수 보도, 시간 경과 동적 갱신, 작전 패턴 감지.
- **§17 학술 근거 2개 추가** — Cohen et al. (2012, 정보 우위 → 가격), Shefrin & Statman (1985, Disposition Effect)

### 변경
- **§6.4 강화** — 분할 매수를 *"전략적 사전 계획"* 으로 명시. 1차 5% / 2차 3% / 3차 2% 단계 명시. *"감정적 물타기"* 와 구분.
- **§6.2 정정** — 동시 보유 *"3~5종목"* → *"5~10종목"* (종목당 10% 한도와 일치)
- **§9 구조 변경** — *"실거래 개입 정책"* 에서 *"실거래 개입 정책 + 트리거 시스템"* 으로 통합. 트리거가 개입 모드와 직결됨.
- **§14 정체성 갱신** — 예측 ❌ 대응 ⭕ + Multi-tier 트리거 명시
- **§10.1 추가 항목** — *"Mode B에서 포지션 늘리기·손절 풀기 ❌"*, *"단독 보도 trust_score 무시 ❌"*

### 정정
- **물타기 개념 정정** — 이전 v3에서 *"잘못된 진입의 결과"* 로 깔린 것을 *"전략적 사전 계획"* 으로 명확화. 매크로 충격이 예측 불가하기 때문에 분할 진입이 필수 전략.

### 영향
- 1주차 데이터 파이프라인 설계 시 **수급 데이터를 가장 우선** 처리해야 함 (Tier 1 트리거).
- LLM 신호 모듈에 **trust_score 출력**을 명시적으로 포함해야 함.
- 백테스트 엔진에 **분할 진입 시뮬레이션** 기능 필요.
