# ADR-0006: 수정주가 보정 (액면분할/병합 back-adjust)

- **상태:** 수락 (분할/병합만 보정 / 배당 미보정)
- **날짜:** 2026-07-22
- **관련:** BORB-30, DATA_SCHEMA.md, ADR-0002(marcap), ADR-0005(케이스 검사기)

## 맥락 (Context)

BORB-30 에서 **marcap 은 원주가(raw)** 로 확정됐다 — 삼성전자 2018-05-04 50:1 분할일에 종가가
2,650,000→51,900 으로 뚝 떨어지고 상장주식수가 ×50, 시총은 연속. 원주가를 그대로 쓰면 분할일이
하루 **−98% 폭락**으로 잡혀 차트가 가짜 절벽이 되고, 그 구간의 수익률·전략이 전부 오염된다.

## 검토한 선택지 (Options)

**A. Stocks 비율로 분할을 감지해 자체 back-adjust.**  ← 채택
- 장점: 외부 소스 없이 marcap 만으로. `Stocks`(상장주식수)가 분할의 정확한 배수를 준다.
- 단점: 휴리스틱(임계값). 배당은 보정 못 한다(총수익 아님). 유상증자/감자 등 경계 케이스 주의.

**B. 외부 수정주가 소스(KIS 등).**
- 장점: 배당까지 반영된 수정주가 가능.
- 단점: 종목별 API 호출 폭발(BORB-34 계열과 같은 문제). 지금 조달 안 됨.

**C. 원주가 그대로 둔다.**
- 단점: 분할 종목이 전부 거짓. 방법론 가드레일(CLAUDE.md: look-ahead·왜곡 금지) 취지에 어긋난다.

## 결정 (Decision)

**A 채택.** `api/main.py` `_split_adjustment()`. 분할 판정 = 두 조건 동시 만족:

1. 상장주식수가 크게 변함 (×≥1.5 또는 ×≤2/3)
2. 종가가 그에 **역방향으로 맞게** 튐 (주식수비 ≈ 가격 역비, 20% 이내)

→ 유상증자(주식수만 늘고 가격은 그만큼 안 빠짐)는 2번이 안 맞아 자동 제외된다.
분할 배수 = `Stocks` 비율(정확값). **최신일 기준** 으로 과거 OHLC 에 계수를 곱해 축소(연속화),
거래량은 반대로 나눠 키운다(비교 가능). **거래대금(Amount)은 분할 불변이라 그대로.**

- 최신일 기준 계수라 Pro 가 스크롤로 구간을 나눠 요청해도(pagination) 이음매 없이 일관된다.
- `/api/candles?adjust=false` 로 원주가도 받을 수 있다. **기본은 보정(adjust=true).**
- 임계값(1.5·2/3·0.2)은 전부 placeholder (CLAUDE.md).

검증: 삼성전자 2018-05 절벽 0.0196(−98%) → 보정 0.9792(−2% 정상), 분할 없는 구간 불변
(`tests/test_api.py`).

## 결과 (Consequences)

- 케이스 검사기 차트가 분할 종목에서 연속으로 보인다.
- **배당은 보정 안 된다** → 이건 총수익(total return)이 아니라 가격수익 기준이다. 배당 포함이 필요하면
  별도 소스/로직이 필요하다(미해결).
- 백테스트가 이 보정을 쓸지, 아니면 원주가+별도 처리를 쓸지는 백테스트 엔진 붙일 때 결정한다.

## 미해결 (Open questions)

1. **배당 보정** — 총수익 기준이 필요하면. 현재 미보정.
2. **임계값 튜닝·검증** — 알려진 액면분할/병합 목록과 대조해 오탐/미탐 점검.
3. **경계 케이스** — 감자, 합병, 대규모 유상증자에서 오작동하지 않는지.
4. **백테스트 사용 여부** — 엔진이 보정가/원가 중 무엇을 쓸지(수익률은 보정가가 맞다).
