# ADR-0001: 종가 진입의 look-ahead 처리

- **상태:** 제안 (오너 입력 대기)
- **날짜:** 2026-07-10
- **관련:** PROJECT_GUIDELINES §6.6(일일 운영 리듬, 종가 매수), §4.1(데이터 3분할), CLAUDE.md(방법론 가드레일)

## 맥락 (Context)

오너의 매매 스타일은 **종가 매수 중심**(음봉 눌림 발목 부근을 종가에 매입, §6.6).
그런데 백테스트에서 이걸 순진하게 구현하면 **구조적 look-ahead**가 생긴다:

> 당일 종가는 장이 마감돼야 확정된다. "당일 종가를 보고 당일 종가에 산다"는
> 미래(마감 결과)를 알고 그 시점에 진입하는 것 = 미래 정보 누설.

이 처리를 정하지 않으면 백테스트 수익률이 낙관 편향되고, 인수인계서가 *"이 처리 없이
넘어가지 마라"* 로 못 박은 항목에 걸린다. 실거래(Phase A)에서 재현 불가능한 알파가 나온다.

## 검토한 선택지 (Options)

**A. 종가 스냅샷 결정, 종가 체결 가정**
- 신호(피처)는 **전일까지** 데이터로 계산 → 오늘 종가에 진입.
- 즉 "오늘 종가에 살지"는 어제 확정. 오늘 데이터는 진입 판단에 안 씀.
- 장점: look-ahead 완전 제거, 구현 단순. 단점: 오너의 "당일 눌림 보고" 직관과 다름.

**B. 마감 직전 스냅샷(예: 15:20) 결정, 종가 체결**
- 15:20 시점 가격/거래대금으로 신호 계산 → 종가 단일가(동시호가)에 진입.
- 장점: 오너 실제 행동("장중 눌림 확인 후 종가 매수")과 가장 근접.
- 단점: 일봉 데이터(marcap)엔 15:20 스냅샷이 없음. **분봉 데이터가 있어야 재현 가능** →
  백테스트 초기엔 구현 불가. 15:20≈종가 근사 가정이 필요.

**C. 당일 종가 신호 → 익일 시가 진입**
- 오늘 종가로 판단 → 내일 시가에 산다. look-ahead 없음.
- 장점: 깔끔. 단점: 오너는 종가 매수지 익일 시가 매수가 아님. 갭 리스크가 결과를 왜곡.

## 결정 (Decision)

**미정 — 오너 입력 필요.** 실거래에서 매수 판단을 실제로 **언제** 내리는지에 달렸다:

- 15:20 이전에 "오늘 종가에 사자"를 확정한다 → **B** (일봉 백테스트에선 A로 근사, 분봉 확보 후 B로 정밀화).
- 종가 확정된 뒤(또는 저녁 회고에서) 판단해 다음 기회에 반영한다 → **A** 또는 **C**.

초기 일봉 백테스트는 **A로 시작**(가장 보수적, look-ahead 0)하고, 분봉 데이터 확보 시 B와
비교하는 것을 잠정 방향으로 제안한다.

## 결과 (Consequences)

- 선택에 따라 신호 계산 시점(피처의 "as-of" 날짜)이 달라진다 → 백테스트 엔진의 진입 로직에 직접 반영.
- 어떤 선택이든 **"신호 계산 시점 < 체결 시점"** 불변식을 코드로 강제하고 테스트로 검증한다.
- 거래비용/슬리피지는 별개 처리(§6.4). 종가 동시호가 슬리피지 가정은 여기가 아니라 비용 모델에서.

## 미해결 (Open questions)

2. 분봉 데이터 소스 확보 여부(B 정밀 구현 조건). KIS 분봉 API 검토 필요.
